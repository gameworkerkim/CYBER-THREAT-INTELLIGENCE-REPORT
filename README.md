## Why This Report Matters — 리포트의 의의

`CTI-2026-0505-VIBE`는 본 아카이브의 다른 단일 사건 분석 리포트들과 달리 **2026년 사이버 위협의 구조 변화 자체를 한 권에 정리한 종합 리포트**다. 다음 여섯 가지 차원에서 본 리포트의 의의가 있다.

### 1. "사이버 = 키네틱 1층"의 공식화 — 최초의 한국어 종합 분석

2026년 1월 트럼프 대통령이 카라카스 정전을 두고 *"우리가 가진 어떤 전문성 때문"* 이라고 공개 발언한 사건은, 미국이 다른 국가에 대한 사이버 공격의 작전 사용을 공개 인정한 첫 사례군에 속한다. 본 리포트는 RUSI·CSIS·SecurityWeek·Schneier on Security·NYT 보도를 교차 검증해 이 변곡점의 정확한 시퀀스(02:00 정전 → 02:01 헬기 진입)와 함의를 한국어 정책 독자가 이해할 수 있는 형태로 정리한 **첫 종합 분석**이다.

### 2. "가난한 나라가 핵에 준하는 파괴력을 갖는 법" — 본 리포트의 독자적 명제

본 리포트는 *Stuxnet(2010) → 우크라이나 정전(2015~16) → 카라카스(2026)* 의 3단 증명이 이미 끝났고, 남은 변수는 **공격 능력의 가격 하락**이라는 명제를 데이터로 입증한다. Cloudflare 2026 보고서의 "사이버 위협의 완전한 산업화" 선언과 Sonatype의 악성 패키지 폭증 통계(2022년 5만5천 건 → 2025년 45만4천 건, 8배)를 결합해, 생성형 AI 구독 한 건이 한 도시의 인프라를 흔들 수 있는 시대의 윤곽을 그렸다.

### 3. 한국이 핵티비즘 1차 표적군임을 실증 — 타임라인 정리

S2W TALON, StealthMole, Cyble의 1차 자료를 종합해 **2025년 3월~2026년 2월 한국 표적화 타임라인**을 정리했다. 경기도지사 사이트(2025-03), 한국캐나다 신문사(2025-03-07), 한국법률연구원(2025-03-24), 육군 훈련소 katc.mil.kr(2026-02-09~11)으로 이어지는 Holy League 동맹 40+ 그룹의 정기 표적화 패턴을 단일 표로 압축해 정책·언론 인용이 가능하게 했다.

### 4. "사이버 둠스데이" 시나리오의 부품 인벤토리

본 리포트는 본인의 별도 리포트 `CTI-2026-0422-MCP`(MCP 잠복 백도어)와 `CTI-2026-0420-VERCEL`(AI SaaS 공급망), 그리고 애플 CVE-2026-20700(dyld 19년 잠복 + WebKit zero-click 체인) 사례를 결합해, **인지 편향 주입 + Sleeper MCP + 약한 취약점 체이닝 + 핵티비즘 디도스 가림막**이라는 4중 결합 시나리오의 부품이 이미 모두 공개돼 있음을 보여준다. 이는 단일 위협 리포트가 아닌 **시나리오 모델링**으로서의 새로운 시도다.

### 5. 한국 정부·기업·시민 대상 실행 가능한 처방 — AI 기반 다중 방어 4축

본 리포트는 진단에 머물지 않고, *AI 이상 탐지 + AI 위협 사냥 + AI 체이닝 분석 + AI 인지 편향 모니터링* 의 4축 다중 방어 체계를 구체적인 정부·기업·시민 수준의 실행 항목으로 분해했다. "AI 사이버 안보 뉴딜"이라는 정책 프레임과 함께 정부24·홈택스·금융결제원·katc.mil.kr 같은 1차 행정·국방 도메인의 다중화 의무화를 명시 권고한다.

### 6. 언론·정책 보고서로 즉시 활용 가능한 Q&A 구조

부록 직전에 배치된 **8개 Q&A 섹션**은 추가 인터뷰나 통화 없이도 일반 독자와 언론이 핵심 그림을 정확히 잡을 수 있도록 설계됐다. 각 답변은 인용 가능한 길이로 작성됐고, 부록 A의 "핵심 인용구 모음"은 기사 제목·리드·발문에 그대로 쓸 수 있는 짧은 인용 후보를 제공한다.

> **요컨대**, `CTI-2026-0505-VIBE`는 사건 보고가 아닌 **시대 진단**이다. 2026년의 사이버 환경이 *"AI를 공격에 쓰는 시대"* 로 이미 이동했음을 입증하고, *"AI를 방어에 쓰는 시대"* 로 옮기는 향후 2~3년의 핵심 과제를 정부·기업·시민 각 층위로 분해한 한국어 정책 자료다.

## Methodology — 분석 방법론

본 아카이브의 모든 리포트는 다음 표준을 따릅니다.

### Traffic Light Protocol (TLP) 분류

| 라벨 | 의미 | 본 저장소 기준 |
| --- | --- | --- |
| **TLP:GREEN** | 커뮤니티 내 공유 가능, 대외 공개 가능 | **이 저장소의 기본값** |
| TLP:AMBER | 조직 내부 공유 한정 | 해당 없음 |
| TLP:RED | 개별 수령자 한정 | 해당 없음 |

### 심각도 등급 (Severity)

| 등급 | 의미 | 대응 시간 |
| --- | --- | --- |
| **CRITICAL** | 국가안보·대규모 민간 피해 직결 위협 | 즉시 |
| **HIGH** | 산업·생태계 광범위 영향 | 24~72시간 |
| **MEDIUM** | 특정 기업·조직군 제한 영향 | 7일 이내 |
| **LOW** | 인지·관찰 수준 | 30일 이내 |

### 신뢰도 평가 (Confidence)

각 Key Judgment는 **High / Medium / Low** 3단계로 신뢰도를 명시하며, 1차 자료와 2차 언론·공개 CTI 자료를 교차 검증합니다.

## Naming Convention — 파일 명명 규칙

```
CTI-YYYY-MMDD-<SUBJECT>[-<SUBTYPE>]_<LANG>.<ext>

예시:
  CTI-2026-0505-VIBE_KR.md          → 2026년 5월 5일 발행, Vibe AI Hacking, 한국어
  CTI-2026-0507-SCARCRUFT_JP.md     → 2026년 5월 7일 발행, ScarCruft, 일본어
  CTI-2026-0422-MCP-PRESS_KR.md     → MCP 리포트의 한국어 보도자료
  CTI-2026-0320-CORUNA_KR.md        → 2026년 3월 20일 발행, Coruna 분석, 한국어
```
