- (HIGH) 2026년 6월 안드로이드 보안 업데이트는 총 **124개의 보안 결함을 패치**하며, 특히 **제로데이 취약점(CVE-2025-48595) 1건을 포함**한다.

현재까지 공개된 정보 기준으로는 다음 사항이 확인되지 않았다.

## §1. 사건 개요 — 안드로이드 프레임워크 제로데이 취약점(CVE-2025-48595) 악용 사례
구글은 2026년 6월 2일(현지 시간), 안드로이드 운영체제의 **2026년 6월 정기 보안 업데이트**를 공개했다. 이번 업데이트는 매년 6월 발표되는 주요 업데이트로, 특히 '안드로이드 프레임워크(Android Framework)'에서 발견된 제로데이 취약점 CVE-2025-48595를 해결하는 데 초점이 맞춰졌다. 구글의 보안 게시판에 따르면, 이번 업데이트는 **2026-06-05** 보안 패치 수준부터 적용되며, 모든 안드로이드 파트너는 패치 발표 최소 한 달 전에 이 문제에 대한 알림을 받았다. 심각도 평가는 개발 목적의 플랫폼 및 서비스 완화 조치가 없는 경우를 기준으로 산정되었다. 구글은 이번 업데이트가 게시된 후 48시간 이내에 안드로이드 오픈소스 프로젝트(AOSP) 저장소에 소스 코드 패치를 공개했다.

## §2. 취약점 상세 분석

#### ① CVE-2025-48595 — 안드로이드 프레임워크 정수 오버플로우 제로데이 취약점
- **특징**: 안드로이드 프레임워크에서 발생하는 정수 오버플로우(Integer Overflow) 취약점이다. 이 결함은 앱이 직접 상호작용하는 API 및 시스템 서비스 집합 내에서 코드 실행을 유발할 수 있다.
- **공격 벡터 및 영향**: 취약점 악용에는 사용자 상호작용이 필요하지 않으며, 기본 앱 권한을 가진 로컬 공격자가 취약점을 이용해 권한을 상승시키고 더 높은 권한 수준에서 코드를 실행할 수 있다. CVSS 점수는 8.4로, '높음(High)' 심각도로 분류되었다.
- **영향 버전**: 안드로이드 14, 15, 16 및 16-qpr2 버전.
- **현황**: 구글은 이 취약점이 '제한적이고 표적화된 공격(limited, targeted exploitation)'에 이미 악용되고 있음을 확인했다. 그러나 공격을 주도한 행위자, 구체적인 표적, 공격 규모 등에 대한 세부 정보는 공개되지 않았다.

## §2-1. Attack Path

공격자는 일반 애플리케이션 권한만 보유한 상태에서도 취약점을 악용하여 시스템 권한 획득이 가능하다.

예상 공격 흐름은 다음과 같다.

Malicious Application ->  Integer Overflow Trigger ->  Framework Memory Corruption -> Privilege Escalation -> System-Level Code Execution

## §3. 패치 및 대응 권고

#### 국내 대응 현황
- 한국인터넷진흥원(KISA)은 구글의 발표 직후 '안드로이드 제품 보안 업데이트 권고'를 공식 발령했다.
- KISA는 국내 사용자들에게 영향받는 안드로이드 OS 버전을 확인하고 제조사 홈페이지를 통해 최신 버전으로 즉시 업데이트할 것을 권고했다.

#### 이용자 대응 요령
**1. 현재 보안 패치 레벨 확인하기**
설정 → 휴대전화 정보 → 소프트웨어 정보 → 'Android 보안 패치 수준' 확인

**2. 즉시 업데이트 적용하기**
해당 항목이 **'2026-06-05'** 이상인지 확인하고, 그렇지 않다면 설정의 시스템 업데이트 메뉴에서 즉시 업데이트를 진행한다.

**3. 구글 플레이 프로텍트 활성화 상태 유지하기**
Google Play 프로텍트는 GMS(Google 모바일 서비스) 적용 기기에 기본 활성화되어 있으며, Play 스토어 외부에서 설치한 앱에 대한 보호 기능을 제공한다.

**4. 출처가 불분명한 앱 설치 자제하기**
CVE-2025-48595는 로컬 권한 상승 취약점으로, 악성 앱 설치 시 공격자가 기기 전체를 장악할 수 있다.
